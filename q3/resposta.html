Questão 3

Os containers em C++ são estruturas de dados que armazenam e gerenciam elementos de um programa. Cada container possui características únicas que os tornam adequados para diferentes tipos de operações.

  1. Lista (List): A lista é uma estrutura de dados que permite inserção e remoção de elementos em qualquer posição. Ela é implementada como uma lista duplamente encadeada. Possui como possíveis uso: gerenciamento de históricos em um navegador, onde é preciso armazenar as páginas visitadas em uma ordem específica.
  2. Fila (Queue): A fila é uma estrutura de dados que segue a política FIFO (First In, First Out), ou seja, o primeiro elemento a entrar é o primeiro a sair. Ela é implementada como uma lista duplamente encadeada ou vetor. Possui como possíveis uso: gerenciamento de tarefas em um sistema operacional, onde as tarefas são executadas na ordem em que são recebidas.
  3. Pilha (Stack): A pilha é uma estrutura de dados que segue a política LIFO (Last In, First Out), ou seja, o último elemento a entrar é o primeiro a sair. Ela é implementada como uma lista encadeada ou vetor. Possui como possíveis uso: gerenciamento de histórico de comandos em um editor de texto, onde as ações realizadas pelo usuário são desfeitas na ordem inversa em que foram realizadas.
  4. Deque (Double-Ended Queue): O deque é uma estrutura de dados que permite inserção e remoção de elementos no início e no final da fila. Ele é implementado como uma lista duplamente encadeada ou vetor. Possui como possíveis uso: gerenciamento de janelas em um sistema operacional, onde as janelas são abertas e fechadas de forma independente.
  5. Vector: O vector é uma estrutura de dados que armazena elementos em um vetor, que é um conjunto de elementos armazenados em posições contíguas da memória. Ele é implementado como um vetor dinâmico. Possui como possíveis uso: armazenamento de dados em um programa, onde é necessário acessar rapidamente os elementos em posições específicas.
  6. Set: O set é uma estrutura de dados que armazena elementos únicos em ordem crescente ou decrescente. Ele é implementado como uma árvore binária de busca balanceada. Possui como possíveis uso: armazenamento de palavras em um dicionário, onde é necessário garantir que não haja palavras duplicadas.
  7. Map: O map é uma estrutura de dados que armazena elementos únicos associados a uma chave em ordem crescente ou decrescente. Ele é implementado como uma árvore binária de busca balanceada. Possui como possíveis uso: armazenamento de informações em um programa, onde é necessário associar um valor a uma chave específica.
